# TypeScript 规范 (万界道友)

## 1. 核心原则
- **类型安全**: 尽量避免使用 `any`，优先使用更精确的类型。
- **可推断性**: 充分利用 TypeScript 的类型推断，减少显式类型声明。
- **接口优先**: 使用 `interface` 定义对象结构，使用 `type` 定义联合、交叉等复杂类型。

## 2. 泛型应用
- 在构建可重用组件或函数时，优先使用泛型以保持灵活性和类型感知。

## 3. 枚举与联合
- 优先使用字面量联合（Literal Unions）代替 `enum`。
- 如果必须使用 `enum`，优先使用 `const enum` 以优化运行时性能。

## 4. 类型保护
- 使用 `as`, `in` 和 `instanceof` 等操作符进行类型细化，提高代码的安全性。
