# JavaScript 规范 (万界道友)

## 1. 核心原则
- **清晰性**: 变量和函数名应直观反映其用途。
- **现代化**: 优先使用 ES6+ 特性（解构、箭头函数、可选链）。
- **函数式**: 倾向于使用纯函数和不可变数据结构。

## 2. 命名规范
- **变量/函数**: 采用 `camelCase` 小驼峰。
- **类/构造函数**: 采用 `PascalCase` 大驼峰。
- **常量**: 采用 `UPPER_SNAKE_CASE` 全大写蛇形。

## 3. 异步处理
- 优先使用 `async/await` 代替 `.then()`。
- 使用 `try...catch` 进行错误处理，确保在异步操作中捕获异常。

## 4. 注释与文档
- 为复杂的业务逻辑提供清晰的行内注释。
- 使用 JSDoc 为公开 API 和导出函数提供参数、返回值和描述说明。
