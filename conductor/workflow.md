# 万界道友 (daoyou.org) - 工作流

## 1. 开发流程 (Development Process)
- **任务驱动**: 所有的开发工作都应围绕 Track 和 Task 展开。
- **测试优先**: 每个功能模块必须有对应的单元测试或集成测试，确保 >80% 的代码覆盖率。
- **原子化提交**: 每个任务完成后，应单独进行 Git 提交。

## 2. 阶段性验证协议 (Phase Verification Protocol)
- **阶段完成**: 每一阶段结束时，必须执行完整的测试套件。
- **代码审查**: 在提交 PR 或合并分支前，确保代码符合已定义的风格指南。
- **部署检查**: 确保本地构建通过后再进行线上部署。

## 3. 记录与总结
- **任务总结**: 在每个任务的提交消息或关联 of Track 文档中提供简短的变更说明。
- **状态更新**: 及时更新 `plan.md` 中的任务状态标记。
