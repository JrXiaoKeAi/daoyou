# 规格说明: 练功房 Bug 修复与 UI 优化 (Training Room Fix & UI Optimization)

## 概述
修复练功房对战单回合停止的问题，并优化页面 UI。新增 100 亿血量木桩，并支持双血条实时显示，对齐标准战斗页面。

## 功能需求
- **对战配置**:
    - 木桩初始 `maxHp` 和 `hp` 设置为 10,000,000,000（一百亿）。
    - 对战固定 10 个回合结束，或玩家手动退出。
- **UI 优化与双血条**:
    - **显示元素**: 页面顶部仿照标准战斗页面，显示“玩家”与“木桩”的实时血量、灵力及其对应的血条进度。
    - **移除图形**: 移除所有自定义的木桩插图或方框。
    - **按钮交互**: “开始演武”按钮显眼，点击后触发对战。
- **日志展示**:
    - 下方为原始战斗日志列表，实时滚动输出。
    - 去除 `font-mono`、`border-ink` 等非标样式，使用与战报页面一致的标准字体。
- **交互逻辑**:
    - 对战时实时更新上方血条数值。
    - 回合结束展示总伤害总结（或通过日志展示）。

## 验收标准
- [ ] 战斗过程中能实时看到玩家和木桩的血条状态及数值变化。
- [ ] 木桩在测试结束前不会因血量耗尽而死亡（100 亿 HP）。
- [ ] 页面风格（边框、字体、间距）与 `/game/battle/[id]` 的战报页面保持高度一致。
- [ ] 按钮和日志滚动逻辑正常工作。

## 超出范围 (Out of Scope)
- 多种类型的木桩选择。
- 历史测试记录持久化。
